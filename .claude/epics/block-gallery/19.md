# Task: TinaCMS collection para metadata de bloques

## Description
Crear una nueva collection en TinaCMS para almacenar metadata de cada bloque (favoritos, deprecados, documentacion custom, etc). Incluye crear los archivos JSON iniciales para los 28 bloques existentes.

## Acceptance Criteria
- [ ] Collection "blocks" definida en tina/config.ts
- [ ] Schema incluye: blockId, displayName, description, category, isFavorite, isDeprecated, documentation
- [ ] Archivos JSON creados para los 28 bloques existentes en content/blocks/
- [ ] Metadata editable desde TinaCMS admin
- [ ] API endpoint para leer/escribir metadata

## Technical Details
- Crear `tina/collections/blocks.ts` con schema
- Agregar collection a `tina/config.ts`
- Script para generar JSON inicial de cada bloque
- Crear `src/app/api/blocks/metadata/route.ts` para CRUD
- Archivos afectados:
  - `tina/config.ts` (modificar)
  - `tina/collections/blocks.ts` (nuevo)
  - `content/blocks/*.json` (nuevos - 28 archivos)
  - `src/app/api/blocks/metadata/route.ts` (nuevo)
