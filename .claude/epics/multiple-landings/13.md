# Task: Extraer templates reutilizables

## Description

Refactorizar `tina/config.ts` para extraer todos los templates de secciones a una constante `sectionTemplates` que pueda ser reutilizada por múltiples colecciones.

## Acceptance Criteria

- [ ] Todos los templates de secciones extraídos a constante `sectionTemplates`
- [ ] La colección `landing` existente sigue usando los mismos templates
- [ ] El home (`/`) sigue funcionando correctamente
- [ ] TinaCMS se regenera sin errores (`npx tinacms build`)

## Technical Details

### Archivo a modificar
- `tina/config.ts`

### Cambios requeridos

1. Identificar todos los templates inline en la colección `landing`:
   - hero, certifications, testimonials, problem, products, ingredients
   - effectiveness, whyChoose, guarantee, howItWorks, finalCta
   - successStories, formulas, activos, videoTestimonials
   - howItWorksNew, finalCtaNew, faq, footerNew
   - heroVideo, stats, ctaTimer, pressLogos, productComparison
   - reviews, beforeAfter, benefits, guaranteeNew

2. Extraer a constante antes de `defineConfig`:
```typescript
const sectionTemplates: Template[] = [
  // Templates existentes ya definidos como constantes
  heroBlock,
  testimonialsBlock,
  // ... templates inline convertidos a constantes
];
```

3. Referenciar en colección landing:
```typescript
{
  type: "object",
  name: "sections",
  list: true,
  templates: sectionTemplates,
}
```

### Verificación
```bash
# Regenerar TinaCMS
npx tinacms build

# Verificar que home funciona
npm run dev
# Navegar a http://localhost:3000
```
